# Skill: daily-report

## Призначення
Створює деталізований Markdown-звіт про роботу над CCTweak Lua Editor.
Зберігається у `docs/YYYY-MM-DD.md`.

## Крок 1: Збір інформації

1. Перевір git-лог сьогоднішніх комітів:
   ```bash
   git -C d:/CCTweak_LuaPageCoding log --oneline --since="1 day ago"
   ```
2. Переглянь git diff останнього коміту щоб зрозуміти деталі змін:
   ```bash
   git -C d:/CCTweak_LuaPageCoding show --stat HEAD
   ```
3. Проаналізуй контекст поточної розмови.

## Крок 2: Форматування звіту

Звіт — тільки **українською мовою**, шаблон:

```markdown
# Звіт — CCTweak Lua Editor
**Дата:** YYYY-MM-DD
**Проєкт:** CCTweak_LuaPageCoding — веб-редактор Lua для ComputerCraft

## Короткий огляд
[1-2 абзаци: що головне зробили, на чому фокус]

## Виконані завдання

### 1. [Назва зміни/фічі]
- **Що зробили:** [опис]
- **Файли:** [які файли змінено]
- **Результат:** [кінцевий стан]

### 2. ...

## Деплой
- [ ] Зміни запушені в master
- [ ] GitHub Actions деплой пройшов успішно
- [ ] Перевірено на bajka.pp.ua/minecraft/lua/

## Наступні кроки
1. [що планується]
2. ...
```

## Крок 3: Збереження

1. Перевір чи існує `docs/` — якщо ні, створи.
2. Поточна дата з контексту системи (`currentDate`).
3. Збережи у `docs/YYYY-MM-DD.md`.
4. Якщо файл за цю дату вже є — **доповни** його новою інформацією.
5. Повідом користувача про шлях до звіту.
