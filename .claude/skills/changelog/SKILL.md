# Skill: changelog

## Призначення
Додає нові клієнтські фічі у `changelog.html` — сторінку "Що нового?" для користувачів Lua Editor.

## Коли використовувати
Після додавання нових фіч, які видно користувачам (UI зміни, нові API, нові режими роботи).
НЕ додавати: внутрішні рефактори, зміни CI/CD, оновлення документації.

## Крок 1: Збір інформації

1. Прочитай поточний `changelog.html` щоб зрозуміти формат та останню версію.
2. Переглянь git-лог щоб знайти нові фічі:
   ```bash
   git log --oneline --since="1 day ago"
   ```
3. Проаналізуй контекст розмови — які зміни стосуються користувачів.

## Крок 2: Визнач що додавати

Відфільтруй тільки **клієнтські зміни**:
- Нові кнопки, режими, UI елементи
- Нові API/моки для симулятора
- Зміни в роботі з серверами (додавання, переключення)
- Покращення UX

НЕ додавай:
- Зміни credentials, env vars, безпеки бекенду
- Рефакторинг коду без видимих змін
- Оновлення документації/звітів

## Крок 3: Додай записи в changelog.html

### Якщо фіча належить до існуючої версії (тої самої дати):
Додай новий `<li>` у відповідний `<ul class="feature-list">`.

### Якщо це нова версія:
1. Змісти `<span class="version-label">Latest</span>` на новий блок
2. Прибери label зі старого блоку
3. Додай новий `<div class="version-block">` перед попереднім

### Формат запису:
```html
<li>
  <div class="feature-icon">EMOJI</div>
  <div class="feature-body">
    <div class="feature-title">Назва фічі</div>
    <div class="feature-desc">Короткий опис українською. Для тегів коду використовуй <span class="feature-tag">назва</span>.</div>
  </div>
</li>
```

### Вибір іконки (HTML entities):
- &#x1F310; — сервери, мережа
- &#x1F4BB; — локальний режим, ПК
- &#x2194; — панелі, розмір
- &#x23E9; — консоль, введення
- &#x2139; — інформація, статус
- &#x26A1; — покращення, швидкість
- &#x1F5A5; — периферія, пристрої
- &#x1F4CB; — шаблони, списки
- &#x1F4D1; — вкладки, файли
- &#x1F3A8; — теми, оформлення
- &#x2601; — хмара, деплой
- &#x1F4C1; — файли, менеджер
- &#x1F512; — безпека
- &#x1F527; — налаштування

## Крок 4: Повідом результат

Скажи користувачу які записи додано і запропонуй запушити.
